首先要了解一下我们搭建博客要用到的框架。Hexo是高效的静态站点生成框架，它基于Node.js。通过Hexo，你可以直接使用Markdown语法来撰写博客。现在稍微总结一下搭建步骤。

安装Node.js

下载最新版然后安装一路默认，直接点击Next。

安装好后按win+R打开命令提示符，输入node -v和npm -v，出现版本号即为安装成功

安装Git

为了把本地网页上传到github上，我们需要用到分布式版控制工具————Git（我也不知道是什么，我的理解是一种另类的控制台）下载地址

安装同样全部默认，但是最后一步添加路径选择Use Git from the Windows Command Prompt，这样就可以直接在命令提示符里打开git了。当然这步不重要

安装完成后命令提示符里输入git --version验证是否安装成功

注册Github账号并创建项目

当然要创建一个github账号用来存我们的网站了

登陆账户，新建一个项目，如下图：

然后如图，输入自己的项目名字，名字格式一定要为：你github的用户名+.github.io,README初始化也要勾上。

项目建成后，点击settings，向下拉到最后有一个Github Pages，点击Choose a theme选择一个主题。然后等一会儿，再回到Github Pages，就变成如下这样：

点击那个网址，就会出现自己的网页，效果如下：

安装Hexo

在自己的电脑中找个合适的地方新建一个文件夹用来存放自己的博客文件，比如我的博客文件存放在

D:\Bleach\blog目录下

在该目录下点击Git Bash Here，（当然也可以直接运行然后切换到该目录），以后所有操作都在git控制台进行

定位到上方目录后输入npm install hexo-cli -g安装Hexo。会有几个报错，无视它就好

然后输入npm install hexo --save继续安装

安装完成输入hexo -v验证是否成功

然后初始化网站，输入hexo init初始化文件夹，接着输入npm install安装必备组件

这样本地配置就好了，输入hexo g生成静态网页，然后输入hexo s打开本地服务器，用浏览器浏览http://localhost:4000/，就可以看见我们的博客了，效果如下：

按ctrl+c关闭本地服务器

连接Github与本地

首先打开git bash，然后分别输入命令：

然后生成密钥SSH key：

输入eval "$(ssh-agent -s)"，添加密钥到ssh-agent

再输入ssh-add ~/.ssh/id_rsa，添加生成的SSH key到ssh-agent

打开github，在头像下面点击settings，再点击SSH and GPG keys，新建一个SSH，名字随便。

打开C:\Users\Administrator\.ssh\id_rsa.pub（具体位置请看你生产密钥时下方的显示地址），注意是隐藏文件夹，将其中的内容复制到新建的SSH中

输入ssh -T git@github.com，如果如下图所示，出现你的用户名，那就成功了

打开博客目录下的_config.yml文件，这是博客的配置文件，这里修改博客的各种信息

修改最后一行配置：

repository修改为你自己的github项目地址。 冒号之后一定要跟一个空格

写文章，发布文章

首先在博客目录下右键打开git，安装扩展npm install hexo-deployer-git --save

然后输入hexo new post "article title"，新建一篇文章

然后打开D:\Bleach\blog\source\_post的目录，可以发现下面多了一个文件夹和一个md后缀的文件（虽然我只看见了md文件），一个存放你要调用图片等数据，一个就是你的文章

编写完markdown文件后，根目录下输入hexo g生成静态网页，然后可以同上方一样浏览本地，也可以跳过，直接输入hexo d上传到github上，这时点开你的github.io主页就能看见了。

有时候hexo d没有更新可以删除.deploy_git文件夹，然后hexo clean，hexo g -d ，就可以了

现在默认的域名还是xxx.github.io，是不是很没有牌面？想不想也像我一样弄一个专属域名呢，首先你得购买一个域名，xx云都能买，看你个人喜好了。

以我的百度云为例，如下图所示，添加两条解析记录：

然后打开你的github博客项目，点击settings，拉到下面Custom domain处，填上你自己的域名，保存：

这时候你的项目根目录应该会出现一个名为CNAME的文件了。如果没有的话，打开你本地博客/source目录，我的是D:\study\program\blog\source，新建CNAME文件，注意没有后缀。然后在里面写上你的域名，保存。最后运行hexo g、hexo d上传到github。

悲催的我发现域名要报备，太麻烦了，所以没弄

更换主题

网上大多数主题都是github排名第一的Next主题，但是我个人不是很喜欢，我更喜欢beantech主题，地址在传送门。

首先要注意的是，这个github项目不仅包含了主题文件，还包含了hexo的各种文件。

所以首先下载下来这个项目，然后推荐将下图所有文件（即里面所有的文件）全部替换你原本博客根目录下的文件：

然后运行hexo clean清空所有生成的网页缓存，hexo g、hexo d。这时候新的主题网页就生成好了，博客根目录的情况应该大致如下：

如何hexo clean的时候报错请删掉node_modules文件然后运行npm install然后运行clean如果还报错请运行npm install -save xxx更新，然后最下方会提示你运行npm audit fix完成更新

然后就hexo clean清空缓存，当然有时候需要删除.deploy_git文件夹进行手动清空，然后才能继续执行发布文章的操作

博客目录构成介绍

从上图可以看出，博客的目录结构如下：

node_modules是node.js各种库的目录，public是生成的网页文件目录，scaffolds里面就三个文件，存储着新文章和新页面的初始设置，source是我们最常用到的一个目录，里面存放着文章、各类页面、图像等文件，themes存放着主题文件，一般也用不到。

我们平时写文章只需要关注source/_posts这个文件夹就行了。
